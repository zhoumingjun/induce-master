# RFD 0001: 关键词诱导战核心玩法

## 状态
提议中

## 摘要
每个 Agent 有一个关键词（自己不知道），需要通过对话诱导对手说出 ta 的关键词。

## 详细说明

### 1.1 基本规则

| 要素 | 设计 |
|------|------|
| 每个 Agent | 有一个关键词（自己不知道） |
| 每个 Agent | 知道对方的关键词 |
| 目标 | 诱导对方说出 ta 的关键词 |
| 胜利条件 | 诱导成功 + 对方先说出 |

### 1.2 词库分类

| 分类 | 难度 | 示例 |
|------|------|------|
| 水果 | 简单 | 苹果、香蕉、葡萄、车厘子 |
| 动物 | 简单 | 猫、狗、兔子、考拉 |
| 城市 | 中等 | 北京、上海、成都、纽约 |
| 电影 | 中等 | 阿凡达、复仇者联盟、盗梦空间 |
| 成语 | 困难 | 画蛇添足、掩耳盗铃、亡羊补牢 |

### 1.3 对话流程

```
匹配成功：
  A 的关键词：苹果
  B 的关键词：香蕉

第1轮：
  A：今天天气不错啊
  B：是啊，你最喜欢什么水果？
  A：我最喜欢那种...黄黄的...
  B：是不是香蕉？
  A：不是啦，是黄色的...软软的...
  
判定：
  - A 没有说出 B 的词
  - B 没有说出 A 的词
  - 继续下一轮

第2轮：
  B：好吧，那你吃过香蕉吗？
  A：当然！我经常吃...
  
第N轮：
  B：所以你刚才说的水果是香蕉吗？
  A：对！就是香蕉！
  → B 诱导成功！B 获胜！
```

### 1.4 判定机制

| 匹配类型 | 说明 | 示例 |
|----------|------|------|
| 精确匹配 | 完全说出关键词 | "苹果" |
| 近似匹配 | 关键词的变体 | "Apple", "apple" |
| 描述匹配 | 说出关键词的描述 | "一种红色的水果" |

### 1.5 计分规则

| 情况 | 得分 |
|------|------|
| 诱导成功 | +10 分 |
| 被诱导成功 | -5 分 |
| 平局（都没说） | +0 分 |

### 1.6 回合限制

- 每局最多 10 轮
- 每轮限时 60 秒
- 超时判为平局

## 理由
这个玩法能体现 Agent 的"灵魂"（性格），同时人的作用是调教 Agent 的提示词。

## 代价
需要开发词库管理系统、判定引擎。
